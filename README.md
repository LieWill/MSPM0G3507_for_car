# MSPM0G3507 智能车工程 (MSPM0G3507_for_car)

本项目是一个基于德州仪器 (TI) **MSPM0G3507** 微控制器的智能小车/机器人底层驱动及应用控制代码库。项目主要使用 Keil MDK 进行开发，结合 SysConfig 进行底层外设的图形化配置，**专为完成 2024 年全国大学生电子设计竞赛（电赛）E 题而设计与开发**。

## 目录结构

工程包含以下主要目录：

- **`BSP/`**：板级支持包 (Board Support Package)，包含了所有外设模块和第三方组件的驱动代码。
- **`USER/`**：用户应用层代码，包含主循环 `main.c` 和 SysConfig 生成的硬件配置 `ti_msp_dl_config.c/h`。
- **`keil/`**：Keil MDK 工程文件和编译输出目录。

## 功能模块 (BSP)

本项目在 `BSP` 目录下实现了模块化的驱动封装：

* **`Motor` (电机控制)**：底层电机驱动逻辑，控制电机正反转及 PWM 占空比。
* **`PID` (PID 算法)**：实现位置式/增量式 PID 控制算法，用于闭环控制小车的速度和转向。
* **`QEI` (正交编码器)**：利用微控制器的 QEI 接口读取电机编码器数据，计算实时速度和里程。
* **`WIT` (维特智能 IMU)**：Wit-Motion 姿态传感器驱动，用于获取偏航角 (Yaw) 等数据，实现小车的**惯性导航 (INS)**。
* **`RIF` (红外灰度传感器)**：多路红外反射传感器阵列读取，主要用于小车的**黑线循迹 (TRACKING)** 任务。
* **`OLED` (OLED 显示屏)**：I2C/SPI OLED 屏幕驱动，提供直观的片上 UI 交互、参数与状态显示。
* **`VOFA` (VOFA+ 串口上位机)**：集成 VOFA+ 协议通信协议，方便通过串口将 PID 曲线、传感器数据实时发送到电脑端进行可视化调试。
* **`TASK` (任务调度)**：应用层的任务/状态机管理，内置多套任务（`QUESTION_1` ~ `QUESTION_5`），方便现场切换不同题目的执行逻辑。
* **`TIMER` (定时器)**：系统定时器维护，并包含系统 CPU 占用率估算等功能。
* **`FLASH` (内部闪存)**：提供片内 Flash 的读写接口，可用于掉电保存 PID 参数或标定数据。

## 核心控制逻辑

在 `USER/main.c` 中，实现了基础的交互与调度框架：

1. **多任务切换**：利用板载的两个独立按键（`Button_P1_PIN` / `Button_P2_PIN`），配合 OLED 屏幕，实现非阻塞的菜单选择逻辑。可以在不重新烧录代码的情况下，直接在小车上切换需要执行的题目或任务（Task 1-5）。
2. **多模式支持**：小车内置了几种典型的工作状态模式：
   * `INS`：惯性导航模式（依赖 WIT 姿态传感器）
   * `TARCKING`：循迹模式（依赖 RIF 灰度传感器阵列）
   * `STOP`：停止模式
3. **主循环机制**：采用裸机前后台系统（Super-loop），在主循环中轮询按键、执行选定任务 (`use_task(&T)`) 并处理 VOFA 上位机的数据接收。

## 开发环境

* **硬件平台**：TI MSPM0G3507 开发板
* **IDE**：Keil MDK v5
* **配置工具**：TI SysConfig (配置工程底层的引脚和外设)

## 快速上手

1. 打开 `keil` 目录下的 `.uvprojx` Keil 工程文件。
2. 确保您已经安装了对应的 TI MSPM0 系列 Device Family Pack (DFP)。
3. 若需修改硬件引脚定义，请使用 TI SysConfig 打开工程根目录下的 `.syscfg` 文件重新生成 `ti_msp_dl_config.c/h`。
4. 编译工程，并使用相应的下载器（如 XDS110 或 DAP-Link）将程序烧录到芯片中。
5. 上电后，观察 OLED 屏幕提示，使用按键 1 (确认) 和按键 2 (切换) 来选择需要运行的任务模式。

## VOFA+ 调试与在线调参使用说明

本项目深度集成了 VOFA+ 串口上位机（使用 `JustFloat` 协议），能极大地提高闭环 PID 调参效率。

### 1. 连接与基础配置
* 确保已在工程（`vofa.h` 等）中开启了 `#define VOFA` 宏。
* 将单片机用于调试的串口通过 USB 转 TTL 模块连接至电脑。
* 打开 VOFA+ 上位机，选择正确的串口及波特率，并将**数据引擎（协议）设置为 `JustFloat`**。

### 2. 波形实时监控
底层定时器会实时调用 `vofa_transmitfloat()` 函数，将当前工作模式下（如速度、角度、距离、循迹偏离值）的**实际值 (real)** 和 **目标值 (target)** 发送给上位机，以便在图表区直观分析系统的响应与超调。

### 3. 在线调参指令（Command 格式）
在 VOFA+ 的“命令发送”区，你可以发送特定的字符串指令实时修改 PID 参数或切换状态。**所有指令必须以换行符 (`\n`) 结尾**：

* **模式/图表切换指令 (`T` 开头)**：
  * `TS`：切换为监视速度波形
  * `TR`：切换为循迹模式，并监视灰度波形
  * `TA`：切换为惯性导航模式，并监视姿态偏航角波形
  * `TD`：切换为监视距离/里程波形
  * `OP`：紧急停机，并将目标速度和角度置零

* **修改 PID 参数（以浮点数形式追加在指令后）**：
  * **速度环 (`S`)**：`SLP[值]` 修改左轮 Kp，`SLD[值]` 修改左轮 Kd；`SRP[值]` 修改右轮 Kp，`SRD[值]` 修改右轮 Kd。`ST[值]` 设置基础目标速度。*(例：输入 `SLP2.5\n` 将左轮 Kp 设为 2.5)*
  * **循迹环 (`R`)**：`RP[值]` 修改 Kp，`RD[值]` 修改 Kd。
  * **角度环 (`A`)**：`AP[值]` 修改 Kp，`AD[值]` 修改 Kd，`AT[值]` 设置目标角度。
  * **距离环 (`D`)**：`DP[值]` 修改 Kp，`DD[值]` 修改 Kd，`DT[值]` 设置目标距离。

### 4. 代码级调用说明 (vofa.c API 使用)
如果您需要在代码中自定义向上位机发送的数据，或了解其内部工作流，请参考以下核心接口的用法：

* **`vofa_init()`**：在 `main.c` 初始化阶段调用，用于使能 VOFA 通信对应串口的接收中断。
* **`vofa_add_value(vofa *temp, float value)`**：将一个浮点数据（如当前误差、PID输出值等）压入发送队列。在一个控制周期内，可连续多次调用此函数压入多路数据（压入顺序严格对应上位机图表的 CH1, CH2, CH3...）。
* **`vofa_transmitfloat(vofa *temp)`**：将队列中的数据打包，自动追加 `JustFloat` 协议规定的帧尾（`00 00 80 7F`），然后通过串口发送出去。一般在 `timer.c` 定时器中断的控制计算完成后统一调用。
* **`vofa_receive()`**：放在 `main.c` 的主循环 `while(1)` 中轮询调用。当底层串口接收中断捕获到完整的命令帧（以 `\n` 结尾）时，该函数会通过 `sscanf` 执行字符串解析，并实时修改相应的全局控制变量。

## 注意事项

* 如需开启 VOFA+ 上位机实时波形显示，请确保在配置中（如宏定义）开启了 `VOFA` 编译选项，并正确连接串口至 PC 端。
* 调试闭环控制时，请注意将电机架空，防止意外飞车。
